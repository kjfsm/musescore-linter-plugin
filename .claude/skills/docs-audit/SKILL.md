# docs-audit

このプロジェクトのドキュメント整合性を横断的にチェックし、重複・矛盾・古い記述・
リンク切れを検出する。検出と修正の **判断は人間**、Claude は横断 read と整理レポート
までを担当する。

## 実行手順

### 1. 引数の解釈

- 引数なし: 全ドキュメントを横断
- `--focus=<file>`: 指定ファイルを軸に、他と何が重複しているかを集中検出
  - 例: `--focus=CLAUDE.md` → CLAUDE.md が README.md と何を重複しているか
- `--links`: リンク切れ（存在しないファイル / セクションアンカー）を集中検出

### 2. 対象ファイルの収集

```
README.md
CLAUDE.md
.claude/rules/*.md
.claude/skills/*/SKILL.md
```

`find . -name "*.md" -not -path "*/node_modules/*" -not -path "*/.turbo/*"` で追加分を拾う。

### 3. Explore agent を起動

`subagent_type: "Explore"` で agent を呼び、各ファイル全文を読む。

検出観点:
1. **重複**: 同じ内容（手順 / チェック項目表 / LintIR 仕様 / コマンド集）が複数ファイルに書かれている箇所
2. **矛盾**: 廃止された API（`ir.staves` / `ir.unresolvedAnnotations`）や古い手順への参照
3. **古い参照**: 削除されたファイルや変更されたセクション名へのリンク
4. **CLAUDE.md の肥大**: README.md や rules/ と被っていて参照に置き換えられる箇所

レポート形式:
- 矛盾は最優先表示
- 「どのファイル × どのファイル × 何の内容」を明示
- 「どちらに寄せるべきか」の推奨を添える
- 600-800 語に収める

### 4. レポートの整形

```
## 🔴 矛盾（最優先）
1. `ファイル:行` に廃止 API への参照が残存 → 対応

## 🟡 CLAUDE.md 削減候補
1. 〇〇セクション（L○-○）→ rules/checker-contract.md と重複

## 🟠 その他重複・古さ
1. ...
```

### 5. 修正の実行（ユーザー承認後）

- 重複削除は Edit ツールで
- セクション移動は元から削除 + 移動先に追記
- リンク化は元の内容を「詳細は [リンク] を参照」に置き換え

## 重要な原則

- **削除する前に必ず移動先で内容を保全する**
- **CLAUDE.md は最小化が至上命題** — 詳細はすべて参照リンクで委ねる
- **rules/ が詳細の正典** — CLAUDE.md に手順の詳細を書かない
